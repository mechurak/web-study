# 02. 렌더링 — SSR / SSG / ISR / CSR, 서버 컴포넌트

> **핵심 질문**
> - "어디서(서버/클라이언트), 언제(빌드 시/요청 시) HTML을 만드는가"로 4가지 방식을 구분할 수 있는가?
> - 서버 컴포넌트(RSC)와 SSR은 다른 개념이다 — 무엇이 다른가?
> - `"use client"`를 붙이는 기준은? 붙이면 무엇을 잃는가?

## 개념 정리

### 렌더링 방식 4가지
| 방식 | HTML 생성 시점 | 적합한 경우 |
|---|---|---|
| SSG | 빌드 시 | (채우기) |
| ISR | 빌드 시 + 주기적 재생성 | (채우기) |
| SSR | 요청 시 | (채우기) |
| CSR | 브라우저에서 | (채우기) |

- [ ] 각 방식이 적합한 실제 서비스 예시 채우기 (블로그, 대시보드, 상품 페이지…)

### 서버 컴포넌트 vs 클라이언트 컴포넌트
- [ ] 서버 컴포넌트에서 할 수 있는 것 / 없는 것 (DB 접근 vs useState)
- [ ] 클라이언트 컴포넌트 경계 설계 — 트리의 어디에 `"use client"`를 두는가
- [ ] 서버 → 클라이언트로 props 넘길 때 제약 (직렬화 가능해야 함)
